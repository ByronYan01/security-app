# Prometheus-App 大屏项目技术与视觉风格研究分析报告

我已对 `prometheus-app` 工程进行了深度分析。该项目是一个成熟的、具有科技感风格的**算力与 GPU 监控大屏**。为了在后续的两个新大屏项目中保持完全一致的视觉风格与交互体验，我对该项目的所有视觉、架构及技术细节进行了汇总：

---

## 1. 技术栈选型

*   **前端核心**: React 19 + TypeScript + Vite 7
*   **样式与排版**: Tailwind CSS v3.4 + Sass
*   **UI 组件库**: Ant Design 6.x (配置为全局 `darkAlgorithm` 暗色算法)
*   **图表渲染**: ECharts 6.x (`echarts` + `echarts-for-react` 进行封装)
*   **数据管理**: 自定义 React Hooks (`useDashboardData`) 进行数据模拟和处理，使用 `axios` 进行潜在的 API 请求。

---

## 2. 视觉设计系统与配色规范 (Design tokens)

为了保证这一系列大屏的**色系完全一致**，我们在后续开发中应严格遵循以下色值配置：

### 基础背景与面板
*   **大屏背景**: 深海军蓝/黑蓝色 `#020619` (`bg-navy-900`)
*   **面板/卡片背景**: 极简透明，或使用半透明深蓝色 `#112240` (`bg-navy-800`)
*   **科技边框与内发光**: `#19B2FF/60`（即带有 `0.6` 不透明度的科技蓝）
    *   边框效果：`border border-[#19B2FF]/60`
    *   内阴影效果：`shadow-[inset_0px_0px_10px_0px_rgba(25,178,255,0.8)]`

### 字体与数字强调
*   **中文字体**: 全局强制使用 `"PingFangSC", "PingFang SC"`
*   **数字特殊字体**: `font-jlinxin`（吉林新字体），专门用于展示核心数值指标，以增强科技感。
*   **核心数值颜色**: 亮青色 `#0AFFFF` / 淡青色 `#B3FFFF`，字号为 `50px`
*   **文字标签颜色**: 纯白色 `#FFFFFF` / 暗灰色 `#9B9DA5`

### 渐变与状态配色
*   **大标题渐变**: 垂直渐变，从 `#ADF7FF`（淡青蓝）到 `#FFFFFF`（白），并带有一层科技感投影 `drop-shadow(0px 0px 10px rgba(173,247,255,0.4))`。
*   **能耗柱状图渐变**: 橙黄色系，从 `#FFA319` 到 `#FFDA47`，背景底色为 `rgba(255, 163, 25, 0.2)`。
*   **温度柱状图渐变**: 青蓝色系，从 `#7CFCFF` 到 `#46A5FF`，背景底色为 `rgba(70, 165, 255, 0.2)`。
*   **分类图表配色**: 采用亮色调配色组：`["#6FC7FF", "#24BA70", "#AB70FF", "#6366FF", "#FFDA47"]`。
*   **特定厂商标识颜色 (Vendor)**:
    *   绿色系: `#3ED99C`（背景 `rgba(62, 217, 156, 0.5)`）
    *   蓝色系: `#46A5FF`（背景 `rgba(70, 165, 255, 0.3)`）
    *   橙色系: `#FA7736`（背景 `rgba(250, 119, 54, 0.4)`）

## 3. 版面设计与标志性科技元素

后续两个大屏项目必须重用或仿照以下标志性设计元素以保证“系列感”：

1.  **顶部 Header 结构**:
    *   中央是 `1px` 高度的 CSS 渐变长线（横穿屏幕，渐变色为淡蓝紫色 `rgba(138, 155, 247, 1)` 到两端透明）。
    *   大标题使用由上到下的冷色调渐变，底部垫有 `header-bottom.svg` 弧形波纹线。
2.  **科技感边角 (CardFrame)**:
    *   四周使用绝对定位的直角线段进行点缀：
        *   左上角: `absolute -top-[1px] -left-[1px] w-[12px] h-[12px] border-l-2 border-t-2 border-[#19B2FF]`
        *   右下角等以此类推。
3.  **图例/卡片角点**:
    *   在特定小面板（如饼图图例项）的外围加上 `1px solid rgba(0, 181, 216, 0.6)` 边框，并在四个角上放置 `w-[3px] h-[3px] bg-[#00b5d8]` 的发光微型正方形。

## 4. 特色组件设计及算法

*   **词云图组件 (`VendorTagCloud.tsx`)**:
    *   未使用第三方重量级词云库，而是**纯手工实现了一个自适应的椭圆螺旋布局算法**。
    *   根据权重将词条分为三级重要度，自动在容器中从中心呈椭圆轨迹螺旋发散计算无重叠的位置。
    *   监听了 Resize 并配合**防抖和节流**机制，在容器尺寸改变时重新动态缩放并排布词条。
*   **环形进度条组件 (`RingGauge.tsx`)**:
    *   基于 ECharts 的 Pie（饼图）和 Gauge（仪表盘）叠加开发。
    *   核心环由 40 个离散的小格子切片拼接而成（已用格子为渐变色，未用格子为暗色 `#1a202c`），具有极强的机械科技感。
    *   使用 ECharts 的 `custom` 渲染类型，手写 Canvas 在环形轨迹的起点和终点渲染发光的微型圆点。
*   **能耗与温度柱状图 (`Top5BarChart.tsx`)**:
    *   在 ECharts 的 Bar 图基础上加了浅色的半透明柱形背景作为底槽。
    *   数值标签没有贴在柱头，而是通过 `labelLayout` 统一固定悬浮在图表顶端水平线 (`y: 6`)。
    *   利用 `ResizeObserver` 动态计算分类轴文字截断宽度，保证屏幕缩放时标签不堆叠。

## 5. 对后续新大屏项目的规划建议

要开始做接下来的两个大屏项目，我建议：
1.  **架构共享**: 采用与 `prometheus-app` 相同的目录结构 (`components/charts`, `components/common`, `styles`, `hooks`)。
2.  **共享公共组件**: 直接复用 `CardFrame`, `PlainCard`, `BaseChart` 以及自定义的 ECharts 自适应方案。
3.  **色彩体系复用**: 将 `tailwind.config.js` 的 `colors.navy`, `colors.cyan`, `colors.vendor` 以及大标题的渐变直接复制到新项目中。
